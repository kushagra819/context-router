## Our defensible novelty (narrow but real)

DEFENSIBLE NOVELTY (narrow but real): (1) Signal composition. We condition routing jointly on four signals — fixed agent ROLE, UPSTREAM confidence propagated as a handoff feature across the pipeline boundary, discrete WORKFLOW STAGE, and a global BUDGET. No single prior work uses exactly this tuple. AutoMix uses self/own-output confidence to escalate WITHIN a cascade (not a cross-agent upstream handoff); Hybrid LLM/RouteLLM/ZOOTER/EmbedLLM use only query features; MasRouter/DAAO use a learned per-query difficulty latent, not an explicit propagated upstream-confidence feature; 'Orchestrating Intelligence' (2026) uses agent confidence but for a learned/flexible MAS, not a fixed typed pipeline. (2) Fixed semantically-typed topology. We assume a KNOWN, fixed Analyzer->Solver->Verifier structure and route the tier per role-slot, which lets us exploit role-specific cost/accuracy asymmetries (e.g., cheap Analyzer, expensive Solver only when upstream confidence is low, cheap Verifier unless stakes high). MasRouter and DAAO instead spend most of their machinery LEARNING the topology/roles, which is a harder but different problem; our fixed-topology assumption is a deployment-realistic simplification they do not target. (3) Confidence as a cross-agent control variable, not an escalation trigger. Using the Analyzer's confidence to set the Solver's tier, and the Solver's confidence to set the Verifier's tier, is a propagated-signal formulation distinct from cascade self-verification (AutoMix) and from query-only routers. WHERE THE CLAIM IS WEAKEST: (a) 'Per-agent / per-role model-tier routing' as a headline is NOT novel — MasRouter (ACL 2025), DAAO (2025), and 'Explainable Model Routing for Agentic Workflows' (CHI 2026) already assign model tiers per agent/role inside workflows, and the latter explicitly lists role+confidence+stage+budget as signals. We must NOT claim to be first at per-agent routing. (b) 'Confidence-aware multi-agent routing' is also taken — 'Orchestrating Intelligence' (2026) is confidence-aware multi-agent routing across multi-scale models. (c) The budget+role+stage signals individually all appear in BAMAS (budget), Hybrid LLM (tunable quality knob), and cascade-routing (stage/sequence). So our novelty MUST be framed as the specific JOINT conditioning + fixed typed pipeline + upstream-confidence-as-handoff, ideally backed by an ablation showing each signal's marginal contribution and a head-to-head vs MasRouter/DAAO. Without that ablation and that head-to-head, reviewers will say we are a special case of MasRouter/DAAO with hand-specified roles.

## Models & datasets used by comparable work (informs our tier choices)

- **RouteLLM (Ong et al.)**: GPT-4/Mixtral-8x7B (and other strong/weak pairs); MT-Bench, MMLU, GSM8K; trained on ~80k Arena battles + ~120k synthetic samples.
- **FrugalGPT (Chen, Zaharia, Zou)**: Commercial APIs (GPT-3/3.5, J1, GPT-Curie, ai21, etc.); HEADLINES, OVERRULING, COQA datasets.
- **AutoMix (Aggarwal, Madaan et al.)**: 5 LLMs (LLaMa2-13B/70B, GPT-3.5/4 era); 5 reasoning/QA datasets including NarrativeQA, QASPER, COQA, etc.
- **MasRouter (Yue et al.)**: Multiple LLM backbones across a heterogeneous pool; benchmarks include MBPP and HumanEval (code), plus math/reasoning (GSM8K/MATH-class) benchmarks reported in the paper.
- **Hybrid LLM (Ding et al.)**: Small/large pairs (e.g., Llama2-class small vs large); MixInstruct and similar instruction/quality datasets with BART-score quality labels.
- **Cascade Routing / 'A Unified Approach to Routing and Cascading' (Dekoninck, Baader, Vechev)**: RouterBench-style model pools; SWE-Bench, MMLU, GSM8K, and other reasoning/coding benchmarks.
- **RouterBench (Hu et al.)**: 11 LLMs (GPT-4, GPT-3.5, Claude-v1/v2/instant, Llama-70B, Mixtral-8x7B, Yi-34B, Code Llama-34B, Mistral-7B, WizardLM-13B); MMLU, MT-Bench, MBPP, HellaSwag, Winogrande, GSM8K, ARC, RAG set.
- **EmbedLLM (Zhuang et al.)**: Large model pool drawn from Hugging Face/Arena leaderboards; routing evaluated on RouterBench-style and leaderboard tasks (MMLU and others).
- **ZOOTER (Lu et al., 'Routing to the Expert')**: Ensemble of open instruct LLMs (Llama-2/Vicuna/WizardLM-class); 26-subset benchmark spanning many domains/tasks.
- **C2MAB-V (Cost-Effective Online Multi-LLM Selection with Versatile Reward Models)**: Pool of commercial/open LLMs on a scheduling cloud + local server; collaborative task suites with cost metadata (reasoning/QA-style).
- **DAAO — Difficulty-Aware Agent Orchestration (in LLM-Powered Workflows)**: GPT-4o-mini, Gemini-1.5-Flash, Llama-3.1-70B, Qwen-2-72B; GSM8K, MATH, HumanEval, MBPP, GAIA, MMLU.
- **Orchestrating Intelligence — Confidence-Aware Routing for Multi-Agent Collaboration across Multi-Scale Models**: Multiple model scales (small-to-large LLMs); standard reasoning/NLP benchmarks (specifics sparse in the preprint).
- **Explainable Model Routing for Agentic Workflows**: Claude Opus 4.5, Gemini 3 Pro/Flash, Llama 4 Maverick, Mistral Small 3.1, GPT-5; VALS, AIME, MATH-500, MMMU, GPQA, LiveCodeBench, SWE-bench.
- **Router-R1 (Zhang et al.)**: Pool described by price/latency/example perf; 7 general and multi-hop QA benchmarks (e.g., NQ, TriviaQA, HotpotQA, 2Wiki, MuSiQue-class).
- **BAMAS — Budget-Aware Multi-Agent Systems**: Heterogeneous LLM pool on MAS reasoning/coding benchmarks (e.g., GSM8K/MATH/HumanEval-class) with per-model cost.

## Summary

Survey of LLM-routing and cascade literature (through 2024-2025, plus closely adjacent 2025-2026 multi-agent work) relevant to a paper on PER-AGENT model-tier routing inside an Analyzer->Solver->Verifier pipeline, where routing signals are agent role, upstream confidence, workflow stage, and budget. The field splits into four lineages. (1) Query-level routers that pick one model per query from cost/quality signals: RouteLLM, Hybrid LLM (Ding et al.), ZOOTER, EmbedLLM, and the RouterBench/LLMRouterBench evaluation harnesses. (2) Cascades that run cheap->expensive models sequentially with a deferral/confidence gate: FrugalGPT, AutoMix, and Dekoninck et al.'s unified routing+cascading. (3) Online/bandit cost-aware multi-LLM selection: C2MAB-V and Router-R1. (4) The most directly competitive and newest lineage: multi-agent / per-agent / per-role routing inside an orchestrated workflow: MasRouter (ACL 2025), DAAO 'Difficulty-Aware Agent Orchestration' (2025), BAMAS budget-aware MAS, 'Orchestrating Intelligence' confidence-aware multi-scale routing (2026), and 'Explainable Model Routing for Agentic Workflows' (CHI 2026). The key finding for positioning: per-query model selection and confidence-gated cascading are thoroughly solved; per-agent/per-role model-tier assignment inside a workflow is NO LONGER an open frontier as of 2025 — MasRouter, DAAO, and the 2026 confidence-aware/explainable agentic routers already assign different model tiers to different agents/roles, and at least two of them already use confidence and budget signals. The genuinely under-occupied niche is the SPECIFIC combination: a fixed, semantically typed Analyzer->Solver->Verifier topology where the routing policy is conditioned jointly on (role identity, the UPSTREAM agent's confidence as a propagated signal, the discrete workflow stage, and a global budget) rather than on a learned per-query difficulty embedding. Most prior multi-agent routers either learn the topology/roles themselves (MasRouter, DAAO) or treat confidence as a within-cascade escalation gate (AutoMix) rather than as a cross-agent handoff signal in a fixed pipeline.
